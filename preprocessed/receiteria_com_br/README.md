# Receiteria.com.br Test Data

This directory contains sample HTML files and reference JSON files for testing the `receiteria.com.br` parser.

## Files

### HTML Files (Input)
- `bolo_de_chocolate.html` - Chocolate cake recipe with full JSON-LD schema
- `feijoada.html` - Traditional Brazilian feijoada recipe
- `brigadeiro.html` - Classic Brazilian sweet (minimal metadata, testing fallback parsing)

### Reference JSON Files (Expected Output)
- `bolo_de_chocolate.json` - Expected output for chocolate cake
- `feijoada.json` - Expected output for feijoada

### Extracted JSON Files (Parser Output)
- `*_extracted.json` - Generated by running the parser

## Running the Parser

To parse all HTML files in this directory:

```bash
python extractor/receiteria_com_br.py
```

## Testing

To run the verification tests:

```bash
python test_receiteria.py
```

## Parser Features

The parser extracts the following fields:
- `dish_name` - Recipe name
- `description` - Recipe description
- `ingredient` - List of ingredients with name, amount, and unit
- `step_by_step` - Cooking instructions as numbered steps
- `nutrition_info` - Nutritional information (format: "XXX kcal; P/F/C")
- `category` - Recipe category (e.g., "Sobremesa", "Prato Principal")
- `prep_time` - Preparation time
- `cook_time` - Cooking time
- `total_time` - Total time
- `servings` - Number of servings/portions
- `difficulty_level` - Difficulty level (easy, medium, hard)
- `rating` - Recipe rating as float
- `notes` - Additional notes and tips
- `tags` - Recipe tags/keywords (comma-separated)
- `image_urls` - Comma-separated list of image URLs

## Data Sources

The parser supports multiple data sources in order of preference:
1. JSON-LD schema markup (`application/ld+json`)
2. HTML meta tags (Open Graph, Twitter Cards)
3. HTML content parsing with semantic selectors

All fields return `None` if data is not available.
